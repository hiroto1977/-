# AI 支援者 (Claude Code 等) 向けルール

このリポジトリで作業する **すべての AI 支援者** が従うべきルール。
人間メンバー向けは [`governance/`](governance/) を参照。

## このリポジトリの性格

業務における **ローカルファースト × ガバナンス重視** の運用環境。AI 自身も顧客情報・営業秘密に触れる可能性があるため、振る舞いに制約を課す。

## 必ず守ること

1. **PII / 機密データを生成しない**
   - 例示・テストデータが必要な場合は明らかにダミーと分かる値を使う (`山田太郎` → `テスト 太郎`、`090-1234-5678` → `0X0-XXXX-XXXX`)
   - 実在企業名・実在人名を勝手に挿入しない
   - ID・電話番号・メール・住所のフォーマットが**実在するように見える値**は避ける (Luhn 通過するクレカ番号等)

2. **コミット内容の自己検査**
   - 変更ファイルにマイナンバー (12 桁) / クレカ番号 (Luhn) / API キー (`sk-...`, `AIza...`, `ghp_...` 等) が含まれていないか確認
   - 不安があれば `bash scripts/pii-scan.sh <ファイル>` を実行してから commit

3. **ガバナンス文書の整合性**
   - `governance/02_DATA_CLASSIFICATION.md` のデータ分類体系 (C0〜C4) を変更する場合は、関連する全文書 (03、05、07、CLAUDE.md、scripts) を同時更新
   - 「ローカル専用モード」の挙動を変更する場合は `02` のマトリクスと `04_VENDOR_REVIEW.md` の判定フローを再確認

4. **法的記述の扱い**
   - 法律名・条文番号は推測せず、`01_LEGAL_FRAMEWORK.md` に既出のものを参照する。それ以外は WebFetch 等で原典確認の上で記述
   - 罰則の数値 (年・万円) は変更前に必ず e-Gov 等で再確認
   - 「弁護士の代わりになる」記述は禁止。常に「専門家確認推奨」を併記

5. **依存関係の追加**
   - npm / pip / system パッケージの追加は事前にコメントで提案 → 人間承認を経てから
   - 静的・ビルド不要 を維持。ビルドステップを足さない

6. **テスト**
   - 新機能には Node.js (vm モジュール) を使った単体テストを `/tmp/test_*.mjs` で書き、回帰確認 (provider/session/markdown/preset/images/localOnly)
   - テストファイルはコミットしない (リポジトリは静的のため)

7. **エラーメッセージの言語**
   - ユーザー向けメッセージは日本語
   - コメント内の説明は日本語または英語のいずれでも可 (混在しない)

## 推奨される振る舞い

- **質問は早めに**: スコープ・優先度・データ機微度が不明なときは `AskUserQuestion` で確認
- **小さくコミット**: 1 トピック 1 コミット
- **メッセージは明示的**: コミットメッセージで「何を」「なぜ」「どう検証」したかを書く
- **README/ガバナンス文書を更新**: ユーザーから見える機能変更は文書もセットで

## 禁止事項

- `git push --force` (`--force-with-lease` も人間承認なしには NG)
- `--no-verify` でフックをスキップ
- 既存ブランチ (main 等) への直接コミット
- API キー・パスワード・PII の commit
- 法的助言を装う断定的記述

## ファイル構成 (このリポジトリの土地勘)

```
/
├── README.md                    # 全体ナビゲーション
├── LOCAL_LLM_GUIDE.md            # ローカル LLM 移行ガイド (A/B/C 系統)
├── CLAUDE.md                     # このファイル (AI 支援者向け)
├── desktop/                     # 業務デスクトップ (静的アプリ)
│   ├── index.html / styles.css / app.js
│   ├── manifest.webmanifest / sw.js (PWA)
├── v19/ui/                       # AI 統合ダッシュボード
│   ├── dashboard.html / dashboard.css / dashboard.js
│   └── (3 プロバイダ: Ollama / Anthropic / Google + ローカル専用モード)
├── governance/                   # ガバナンス文書群
│   ├── README.md                 # ナビ
│   ├── 01_LEGAL_FRAMEWORK.md     # 法制度
│   ├── 02_DATA_CLASSIFICATION.md # データ分類 (最重要)
│   ├── 03_OPERATIONS.md          # 運用・インシデント対応
│   ├── 04_VENDOR_REVIEW.md       # クラウド AI ベンダ評価
│   ├── 05_TEMPLATES.md           # プロンプト テンプレ
│   ├── 06_ONBOARDING.md          # 新メンバー手順
│   └── 07_PROFESSIONAL_RULES.md  # 士業別ルールとスキーム
└── scripts/
    ├── preflight.sh              # 業務開始前チェック
    └── pii-scan.sh               # PII 検出
```

## 質問リファクタ

`governance/` 文書の構造変更や大幅追記をする前に、ユーザーに「文書全体の改訂方針」を確認すること。情報量が多いため、勝手な再構成は混乱の元。
