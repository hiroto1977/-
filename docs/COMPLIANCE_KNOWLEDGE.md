# 法務・税務・労務 知識ベースと確証プロセス（恒久運用）

Service Hub が参照する法務・税務・労務の制度情報は、誤りがあると利用者の不利益に直結する
ため、**複数の独立した媒体で裏が取れた情報だけ**を採用します。本書はその恒久的な確証
プロセスと、現在の確証済みデータの出典を記録します（[docs/COMPLIANCE_RULES.md](./COMPLIANCE_RULES.md)
の予防ルールを、一次情報で裏取りした制度知識で支える層）。

> ⚠️ 本データは一般的な制度情報であり、**法務・税務・労務の助言ではありません**。最新の
> 適用関係・個別判断は各専門家（弁護士／税理士／社労士）と一次情報で確認してください。

## 原則
1. **多媒体照合**: 1 つの事実につき、国の所管省庁（国税庁・厚労省・個人情報保護委員会 等）・
   政府広報・報道／専門団体など、**独立した出典を 2 件以上**集める。過去の改正経緯も確認する。
2. **公的裏付け必須**: 2 件のうち **1 件以上は公的**（政府・自治体）であること。
3. **確証のとれない情報は採用しない**: 条件を満たさない情報は `unconfirmed` として **破棄**する
   （`sourceVerification.ts` の確証機構が強制し、`complianceResearch.ts` が discarded に数える）。
4. **安全のための人手レビュー**: 収集・取り込みは AI が自動で確定せず、必ず **PR レビュー**を通す。

## 仕組み（コードで強制）
- `src/renderer/data/sourceVerification.ts` — 確証エンジン（独立出典数・公的有無で判定）。
  既定方針 = 独立 2 出典以上・うち公的 1 件以上。
- `src/renderer/data/complianceKnowledge.ts` — 確証済みデータ（制度事実＋出典 URL）。分野は
  税務 / 労務 / 法務（`ComplianceDomain`）。
- `src/renderer/data/complianceResearch.ts` — リサーチ・ループ。全件を検証し、確証件数・破棄件数・
  分野別網羅・**確証ゼロの分野（findings）** を集計。テスト（`complianceResearch.test.ts`）で
  「全エントリ confirmed」「全分野に確証 1 件以上（findings 空）」「未確証は破棄」を**不変条件**化。

## 恒久運用フロー（再実行可能）
1. **収集**: Web/検索で対象制度の一次情報（所管省庁）を調べる。
2. **照合**: 省庁・政府広報・報道／専門団体など独立媒体で同一事実を確認（改正経緯も）。
3. **記録**: 確証できた事実のみ `complianceKnowledge.ts` に出典つきで追加。未確証は破棄。
4. **検証**: `npm test`・`npm run verify:all` で整合を確認。
5. **レビュー**: PR を作成し人がレビュー→マージ。これを定期的に繰り返す（恒久的な更新）。

> 環境上の注意: 実行環境は揮発的で、完全自動・常時稼働の Web クローリングは行いません。
> 「永続的に精度を高め続ける」は、上記の**再実行可能な確証フロー＋人手レビュー**として運用します。
