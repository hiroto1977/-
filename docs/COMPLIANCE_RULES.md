# 法令を踏まえたトラブル防止ルール（コンプライアンス規程）

> ⚠️ **重要な免責**: 本書は Service Hub の開発・運用におけるトラブル（法令違反・権利侵害・
> 利用者被害）を予防するための**社内ルールの叩き台**であり、**法的助言ではありません**。
> 最終的な適法性判断は、対象分野ごとに**弁護士・各士業（税理士／社労士／弁理士 等）・
> 個人情報保護や金融の専門家のレビュー**を受けてください。本書は「どの機能にどの法令が
> 関わり、どんな予防ルールを置き、それが技術的にどこまで担保されているか」を整理し、
> レビューを依頼しやすくすることを目的とします。
>
> 関連: [docs/SECURITY.md](./SECURITY.md)（脅威モデル）/ [docs/DATA_PROTECTION.md](./DATA_PROTECTION.md)
> （データ保護監査）/ [docs/specs/05_AI_COUNSELOR.md](./specs/05_AI_COUNSELOR.md)（AI カウンセラー安全要件）/
> [docs/outsourcing/06_SECURITY_REQUIREMENTS.md](./outsourcing/06_SECURITY_REQUIREMENTS.md)（委託先要件）。

---

## 0. 基本原則（すべての機能に共通）

| # | 原則 | 趣旨 |
|---|---|---|
| G1 | **適法・善管注意** | 関係法令・各サービスの利用規約（ToS）を遵守し、善良な管理者の注意で運用する。 |
| G2 | **データ最小化・ローカルファースト** | 必要最小限の情報のみ取得・保持。秘密情報・個人データは端末内に暗号化保管し、サーバ集約しない。 |
| G3 | **実行を伴わない安全評価** | セキュリティ検証は自分の資産／許可された範囲のみ。他者システムへの実攻撃・走査はしない。 |
| G4 | **資格を要する行為をしない** | 法律で資格者に限定された行為（法律事務・税務代理・投資助言・医行為 等）はアプリが代行しない。情報提供と有資格者への橋渡しに留める。 |
| G5 | **誤認させない表示** | 効果・関係性（広告／アフィリエイト）・専門性について、利用者を誤認させる表示をしない。免責を明示する。 |
| G6 | **人のレビューを通す** | 安全・法令に関わる検知ルール・既定値・文言の変更は、AI が独断で行わず **PR レビュー**を通す。 |

## 1. 個人情報・プライバシー（個人情報保護法 / 要配慮個人情報 / 越境移転）

**対象機能**: Emotions（気分・感情ログ）、AI カウンセラー、Security（メール漏洩照会 HIBP）、
チーム管理（メンバー情報）、各 SaaS 連携で扱う氏名・連絡先。

| ルール | 内容 | 担保 |
|---|---|---|
| P-1 | **要配慮個人情報の特別扱い**: 心身の状態・感情・健康に関する記録（Emotions／カウンセリングのノート）は要配慮個人情報に準じて扱い、取得は本人の操作起点に限り、端末内に暗号化保管する。第三者提供・学習用送信をしない。 | 設計（ローカル保管）／要レビュー |
| P-2 | **暗号化保管**: 秘密情報・個人データは WebCrypto AES-GCM-256 / OS キーチェーンで暗号化。`src/renderer/data` の業務レコードは暗号化方針に従う。 | 一部実装（[DATA_PROTECTION.md](./DATA_PROTECTION.md) の改善項目を継続） |
| P-3 | **目的外利用・第三者提供の禁止**: 取得した個人データを本来の表示・連携目的以外に使わない。外部サービス（Cloudflare Worker プロキシ等）を経由する場合は経路と目的を明示。 | 運用ルール |
| P-4 | **越境移転の確認（APPI 28 条）**: CORS 回避プロキシ等で個人データが国外に渡りうる場合、移転先・同等性・本人同意の要否を確認する。 | 要レビュー（[PROXY_EXAMPLE.md](./PROXY_EXAMPLE.md) 利用時） |
| P-5 | **テストに実個人情報を使わない**: テスト・デモは合成データのみ（実在の氏名・連絡先・SHA を使わない）。 | 運用ルール |
| P-6 | **漏洩時対応**: 個人データ漏洩のおそれを認知したら、速やかに影響範囲を確認し、必要に応じ個人情報保護委員会・本人への通知を検討する（[§7](#7-インシデント対応)）。 | 運用ルール |

## 2. セキュリティ検証と不正アクセス（不正アクセス禁止法 / 刑法 234 条の 2 ほか）

**対象機能**: Security（HIBP/VirusTotal）、レッド×ブルー検知精度ハーネス（`securityRange`）、
URL スキャン、Ollama。

| ルール | 内容 | 担保 |
|---|---|---|
| S-1 | **実攻撃をしない**: 攻撃手法の評価は**実行を伴わない文字列照合のみ**。他者のシステムへ攻撃・脆弱性走査・認証回避を行わない。 | **実装で担保**（`src/shared/securityRange.ts` は攻撃コードを実行せずメモリ内照合のみ） |
| S-2 | **サブプロセス・任意実行の禁止**: ランタイムから任意コマンド／サブプロセスを起動しない。 | **実装で担保**（`lint:forbidden` が `child_process` exec/spawn・`eval`・`new Function` 等を CI で拒否） |
| S-3 | **業務妨害をしない（DoS 防止）**: 外部 API へ過剰なリクエストを送らない。レート制御・キャッシュを用い、相手サービスの規約上の上限を超えない。 | 運用ルール／一部スナップショット設計 |
| S-4 | **自分の資産・許可範囲のみ**: HIBP は自分のメール、VirusTotal は自分が扱う URL など、本人に正当な関係がある対象に限る。 | 運用ルール |
| S-5 | **検知ルールは人がレビュー**: 危機検知・脅威検知のマーカー／既定値の変更は PR レビュー必須。 | 設計（[specs/05](./specs/05_AI_COUNSELOR.md) §7） |

## 3. 資格業務（弁護士法 72 条・税理士法・各士業の独占業務 / 医師法 / 金融商品取引法）

**対象機能**: 士業連携（税理士／社労士／弁護士／司法書士／行政書士／中小企業診断士／弁理士）、
AI カウンセラー、投資（株式／投資信託／不動産投資）、税務試算、資金調達レーダー。

| ルール | 内容 | 担保 |
|---|---|---|
| Q-1 | **非弁・非税理士行為をしない**: 法律事務（弁護士法 72 条）・税務代理／税務書類作成（税理士法 52 条）等、有資格者の独占業務をアプリが代行しない。提供は一般的情報・整理・**有資格者への橋渡し**に留める。 | 設計（士業はスナップショット情報のみ・実務代行なし）／要レビュー |
| Q-2 | **投資助言業をしない（金商法）**: 株式・投信・不動産の機能は**情報提供のみ**で、個別銘柄の助言・投資判断の代行をしない。「投資判断は自己責任」「本機能は投資助言ではない」旨を明示。 | 文言（免責表示）／要レビュー |
| Q-3 | **医行為・診断をしない（医師法）**: AI カウンセラーは診断・治療・医療広告をせず、すべての応答に医療免責を付す。危機検出時は確証済みの公的相談窓口へ確実に誘導する。 | **実装で担保**（[specs/05](./specs/05_AI_COUNSELOR.md)：免責付与・危機→窓口照会 100% をテストで固定） |
| Q-4 | **税務試算は概算・参考**: 税務試算は概算で、納付・申告は公式ツール／税理士へ。「正確な税額・申告は専門家へ」と明示し、納付は手動。 | 設計（公式導線・納付は手動）／文言 |
| Q-5 | **資格・出典の真偽確認**: 紹介・提示する制度・窓口・専門家情報は、確証のとれたもの（独立 2 出典以上・うち公的 1 件以上）のみ採用。 | **実装で担保**（`src/renderer/data/sourceVerification.ts`・[COUNSELOR_KNOWLEDGE.md](./COUNSELOR_KNOWLEDGE.md)） |

## 4. 取引・表示・広告（特定商取引法 / 景品表示法 / ステルスマーケティング規制 / 資金決済法）

**対象機能**: EC（Shopify/BASE/Amazon 等）、フードデリバリー（Uber Eats／出前館）、
アフィリエイト（A8.net／Amazon アソシエイト）、AI ブログくん、Stripe 決済。

| ルール | 内容 | 担保 |
|---|---|---|
| T-1 | **優良誤認・有利誤認をしない（景表法）**: 売上・成果・効果の表示は実データに基づき、誇大表示をしない。サンプル／スナップショットは「サンプル値」と明示。 | 文言（各ページに snapshot 明示）／運用 |
| T-2 | **広告関係の明示（ステマ規制 2023）**: アフィリエイト・PR を含む発信は、広告である旨を明瞭に表示する。AI 生成コンテンツに広告関係を埋め込む場合も同様。 | 運用ルール／要レビュー |
| T-3 | **特商法の表示**: EC・役務提供に関わる表示義務（事業者表示・返品条件等）は、実際の取引主体（出店者）の責任範囲を尊重し、アプリは表示を歪めない。 | 運用ルール |
| T-4 | **カード情報を保存しない（資金決済法 / PCI-DSS）**: 決済はプロバイダ（Stripe 等）に委ね、カード番号等の機微な決済情報をアプリで保存・中継しない。 | 設計（Stripe はトークン／集計のみ）／要レビュー |

## 5. 第三者サービス連携・自動化（著作権法 / 不正競争防止法 / 各サービス ToS / API 規約）

**対象機能**: 全 SaaS 連携、スクレイピング系（公開 API のない NETSEA／スーパーデリバリー／ココナラ等）、
コネクター／自動化、外部リンク。

| ルール | 内容 | 担保 |
|---|---|---|
| C-1 | **アクセス制御を回避しない（不正競争防止法）**: ログイン回避・技術的保護手段の回避をしない。公式 API・OAuth の正規フローのみ用いる。 | **一部実装で担保**（`oauth.ts` は正規 PKCE フロー） |
| C-2 | **ToS・API 規約の遵守**: 各サービスの利用規約・レート制限・取得可能範囲を守る。規約で禁止されたスクレイピングをしない（公開 API のないサービスはスナップショット表示に留める）。 | 設計（snapshot-only サービス群） |
| C-3 | **著作権・データベース権の尊重**: 取得したコンテンツ・サムネイルは表示・連携の範囲で用い、無断再配布・改変をしない。 | 運用ルール |
| C-4 | **外部遷移は OS ブラウザ経由**: 外部リンクは `window.serviceHub.openExternal` 経由（`window.open` 禁止）で、フィッシング誘導・資格情報の誤送信を防ぐ。 | **実装で担保**（`lint:forbidden` 相当の規約・[CLAUDE.md](../CLAUDE.md) 規約） |
| C-5 | **悪意あるサイトへの資格情報送信防止**: 保存先 URL の取り違えで別サイトへ秘密を送らせない（[SECURITY.md](./SECURITY.md) A5）。 | 設計／要レビュー |

## 6. 通信の秘密・労務（電気通信事業法 / 労働関係法令）

| ルール | 内容 | 担保 |
|---|---|---|
| M-1 | **通信の秘密を侵さない**: Gmail／Slack／LINE／Discord 連携で取得したメッセージを、表示・本人の操作目的を超えて解析・蓄積・第三者提供しない。 | 設計（ローカル表示）／運用 |
| M-2 | **労務情報は参考・有資格者へ**: 社労士連携・給与計算（payroll）の機能は概算・整理に留め、社会保険手続等の専門業務は社労士へ橋渡し。 | 設計／文言 |

## 7. インシデント対応

1. **検知**: 法令違反・権利侵害・個人データ漏洩のおそれを認知したら、作業を止めて影響範囲を特定する。
2. **報告**: 24 時間以内に責任者へ第一報（[outsourcing/06](./outsourcing/06_SECURITY_REQUIREMENTS.md) §6 と整合）。
3. **是正**: 暫定対応（公開停止・無効化・ローテーション）→ 恒久対応を PR で実施。
4. **対外対応**: 必要に応じ、個人情報保護委員会・本人・関係当局・対象サービスへの通知を専門家と検討。
5. **記録**: 原因・対応・再発防止を Issue/PR に記録する。

## 8. 専門家レビューが必要な事項（チェックリスト）

締結・公開・本番運用の前に、対象分野の専門家レビューを受けること。

- [ ] 個人情報の取得・保管・越境移転の適法性（弁護士／個情委ガイドライン）— §1
- [ ] 要配慮個人情報（感情・健康）の取扱い同意・最小化 — §1, §3 Q-3
- [ ] セキュリティ機能が不正アクセス禁止法・業務妨害に触れないこと — §2
- [ ] 士業の独占業務・非弁／非税理士行為に当たらないこと（各士業）— §3
- [ ] 投資情報が金商法の投資助言業に当たらないこと（証券・金融の専門家）— §3 Q-2
- [ ] AI カウンセラーが医師法・医療広告に触れないこと — §3 Q-3
- [ ] 表示・広告が景表法・特商法・ステマ規制に適合すること — §4
- [ ] 各サービス ToS・API 規約・スクレイピング可否の確認 — §5
- [ ] 決済まわりの資金決済法・PCI-DSS 適合 — §4 T-4

## 9. 運用（このルールの育て方）

- 本書は機能追加（新サービス・新分野）のたびに該当法令とルールを追記し、**PR レビュー**を通して更新する。
- 既に技術的に担保されているルール（S-1/S-2、Q-3/Q-5、C-4 等）は、対応する CI ゲート・テストが
  green であることをもって継続を確認する（[CLAUDE.md](../CLAUDE.md) の verify/lint 群）。
- 「担保＝設計／文言／運用ルール／要レビュー」の区分は、実装で機械的に保証されているか、
  人手の運用に依存するかを表す。**要レビュー**項目は本番化前に専門家確認を必須とする。
