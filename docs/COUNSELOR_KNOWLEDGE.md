# AIカウンセラー 知識ベースと確証プロセス（恒久運用）

AIカウンセラー（`src/renderer/data/counseling.ts` ほか）が提示する相談窓口・統計・制度の
情報は **人命に関わる**ため、**複数の独立した媒体で裏が取れた情報だけ**を採用します。
本書はその恒久的な研究・確証プロセスと、現在の確証済みデータの出典を記録します。

## 原則
1. **多媒体照合**: 1 つの情報につき、国（厚生労働省 等）・自治体（都道府県/市区町村）・
   運営団体・報道など、**独立した出典を 2 件以上**集める。過去データ（改定前の番号・廃止
   窓口）も確認し、最新で正しいものに揃える。
2. **公的裏付け必須**: 2 件のうち **1 件以上は公的**（政府・自治体）であること。
3. **確証のとれない情報は採用しない**: 条件を満たさない情報は `unconfirmed` として
   **破棄**する（`sourceVerification.ts` の `filterConfirmed` が機構として強制）。
4. **安全のための人手レビュー**: 収集・取り込みは AI が自動で確定せず、必ず **PR レビュー**
   を通す。安全クリティカルな検知ルールや窓口情報を無検証で自動更新しない。

## 仕組み（コードで強制）
- `src/renderer/data/sourceVerification.ts` — 確証エンジン。`EvidenceSource`（出典＋種別）、
  `verifyClaim`（独立出典数・公的有無で confirmed/unconfirmed 判定）、`filterConfirmed`
  （未確証を除外）。既定方針 = 独立 2 出典以上・うち公的 1 件以上。
- `src/renderer/data/counselorKnowledge.ts` — 確証済みデータ（窓口＋出典 URL）。
  テスト（`sourceVerification.test.ts`）で「全窓口が confirmed」「公的出典 1 件以上」
  「実際に提示する `SUPPORT_RESOURCES` と一致」を**不変条件**として固定。

## 恒久運用フロー（再実行可能）
1. **収集**: Web/検索（検索特化型 AI を含む）で対象情報を調べる。
2. **照合**: 国・自治体・運営団体・報道など独立媒体で同一の事実を確認（過去データも）。
3. **記録**: 確証できた情報のみ `counselorKnowledge.ts` に出典つきで追加。未確証は破棄。
4. **検証**: `npm test`（確証テスト）・`npm run verify:all` で整合を確認。
5. **レビュー**: PR を作成し人がレビュー→マージ。これを定期的に繰り返す（恒久的な更新）。

> 環境上の注意: 本リポジトリの実行環境は揮発的で、完全自動・常時稼働の Web クローリングは
> 行いません。「恒久的に継続」は、上記の**再実行可能な確証フロー＋人手レビュー**として運用します。
