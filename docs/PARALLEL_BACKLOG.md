# 並列作業キュー & 待ち時間プロトコル

> 複数の継続タスク（ループ）を、CI 待ち時間を無駄にせず順次・前倒しで進めるための仕組み。
> セッション/`/loop`/「作業を再開」のたびに本キューの先頭から次アクションを実行する。

## 制約（正直な前提）

- 開発ブランチは **`claude/modest-noether-knlmJ` の1本**に限定（タスク指示）。
  複数 PR を同時に開くと差分が混ざるため、**PR は順次**（1本マージ→次を push）。
- 別ブランチ/worktree での真の同時並列は、**ユーザーの明示許可がある場合のみ**
  （`NEVER push to a different branch without explicit permission`）。

## 待ち時間プロトコル（CI 待ちの〜2分を活用）

現 PR の CI を待つ間、**リポジトリを変更しない読み取り作業**でキューの次アイテムを前倒しする:

1. **調査の先行**（コミット不要）: 確証ループ①なら次の制度の一次情報を WebSearch で裏取り、
   コード系②③なら対象コードを読み設計を確定。
2. **マージ後に即コミット**: 現 PR がマージされたら main へ同期し、前倒しで用意した変更を
   すぐコミット→push→PR。アイドル時間を最小化する。
3. **コミットは現 PR マージ後のみ**（単一ブランチを汚さない）。CI 待ち中はローカル編集も避け、
   読み取り/調査に限定する。
4. 真に同時並列が必要になったら、ユーザーに別ブランチ/worktree の許可を求める。
