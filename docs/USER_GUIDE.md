# Service Hub ユーザーガイド

事業オーナー・企画・マーケター向けの「はじめての一冊」です。エンジニアでなくても安心して読めるよう、専門用語は最小限にしています。

---

## 🚀 はじめに

Service Hub は、GitHub・Notion・Slack・Google Drive・Canva・Uber Eats・出前館・株式投資・不動産投資・投資信託・ココナラ・税務試算など **64 種類のサービス**を **1 つのダッシュボードにまとめる業務支援ツール** です。フードデリバリー事業 (Uber Eats / 出前館) と投資事業 (株式投資 / 不動産投資 / 投資信託) の業務操作にも対応しています。
ログイン情報を毎回 Web サイトに入力したり、タブを行き来する手間がなくなります。

主にできること:

- 各サービスの「最近の動き」(投稿、PR、メール、予定など) を 1 画面で確認
- プレゼン、名刺、SNS 画像などをワンクリックで生成 → ライブラリに保管
- 営業チームの強み・弱みを 5 軸レーダーチャートで可視化
- 事業ごとの売上・利益・損益分岐点 (KPI) のチェック

## 📥 起動する 2 つの方法

Service Hub には 2 つの起動方法があります。**初めての方は「方法 A: ブラウザ版」を強くおすすめします。**

### 方法 A: ブラウザ版 HTML (推奨、もっとも手軽)

ダウンロードした 1 つの HTML ファイル (`service-hub.html`、約 400 KB) をダブルクリックするだけで起動します。インストール不要、管理者権限も不要です。

操作の流れ:

1. 配布元から `service-hub.html` を入手し、デスクトップなど分かりやすい場所に置く
2. ファイルをダブルクリック → 普段お使いのブラウザ (Chrome / Edge / Safari / Firefox 推奨) で開く
3. 初回はマスターパスワードの設定画面が表示されます

ブラウザ版でできないことは特にありません。63 サービスすべて利用できます。ただし一部のサービス (Notion / Atlassian / Cloudflare) はブラウザのセキュリティ制限により **CORS プロキシ** を 1 度だけ用意する必要があります (後述の「🌐 CORS で動かないサービス」を参照)。

### 方法 B: Electron デスクトップアプリ

Mac / Windows / Linux 向けのインストーラを使う方法です。OS の鍵管理 (Keychain / 資格情報マネージャー) と連携できるため、企業 IT の規定でブラウザ単体では難しい場合に向いています。

各 OS でのインストール:

- **macOS**: `Service Hub.dmg` を開いて Applications フォルダにドラッグ
- **Windows**: `Service Hub Setup.exe` をダブルクリックしてインストーラに従う
- **Linux**: `service-hub.AppImage` に実行権限を付けて起動、または `.deb` / `.rpm` をインストール

起動後の画面はブラウザ版と同じです。違いは「**トークン(API キー)が OS の鍵管理に保管される**」という点だけです。

開発者向けに、ソースコードから起動するコマンドも用意されています:

```sh
npm install
npm run dev
```

## 🔐 初回セットアップ — マスターパスワード + リカバリーキー

Service Hub を初めて起動すると、**マスターパスワード** の設定画面が表示されます。
このパスワードは「API キー (トークン) を暗号化して守る鍵」です。Service Hub がインターネットに送信することは一切ありません。

### パスワードの選び方

- **8 文字以上** が必須です (実用上は 12 文字以上を強く推奨)
- 短い英単語の組み合わせ (例: `morning-coffee-blue-cat`) は覚えやすく安全
- ご自分の誕生日や `password123` のような推測されやすいものは避けてください
- 失念しにくいことを最優先に。書き留めるなら金庫やパスワードマネージャーへ

### リカバリーキー (24 単語) を必ず保存してください

パスワードを設定すると、続けて **24 個の英単語** が画面に表示されます (例: `abandon ability ... art`)。これが「リカバリーキー」です。

> パスワードを忘れた場合、この **24 単語だけ** がデータを取り戻す唯一の手段です。

推奨される保管方法 (2 つ以上を併用):

- **紙に手書きして金庫やデスクの引き出しに保管**
- **1Password / Bitwarden などのパスワードマネージャー** に「Service Hub リカバリーキー」として保存
- **印刷して封筒に入れ、信頼できる人へ預ける**

画面下部の `📋 クリップボードにコピー` ボタンを使うと:

- 24 単語をコピーできますが、**30 秒後に自動でクリップボードから消えます** (誤って別の場所に貼り付けないようにするための安全機能です)
- コピーできたら速やかにパスワードマネージャーや書き留めに使ってください

`💾 .txt でダウンロード` を使うと:

- ダウンロードフォルダに小さなテキストファイルが作られます
- **ファイルはダウンロードフォルダに置きっぱなしにせず**、速やかにパスワード保管庫または印刷物へ移し、元のファイルは削除してください

「記録完了 — Service Hub を開始」ボタンは、保存できたとチェックを入れない限り押せません。**この画面は二度と表示されない** ので、必ず作業を終えてから進んでください。

## 🔑 トークン (API キー) を入れる

各サービスを「ライブ取得 (実データ取得)」モードで使うには、サービスごとのトークンが必要です。
ここでは GitHub を例に流れをご説明します。

### 例: GitHub の Personal Access Token

1. ブラウザで [https://github.com/settings/tokens](https://github.com/settings/tokens) を開く
2. `Generate new token (classic)` をクリック
3. 名前 (例: `Service Hub`) と権限 (`repo` / `read:user` など) を選び発行
4. 表示された文字列 (例: `ghp_AbCd…`) をすべてコピー
5. Service Hub の **GitHub ページ → トークン入力欄** に貼り付け
6. 「保存」を押す → 自動的に最新データの取得が始まります

トークンは暗号化されて保管され、再起動後もそのまま使えます。

### サービスごとの認証方式

サービスによって入力するものが少しずつ違います。大きく 3 種類です。

| 種類 | 該当サービス | 入力するもの |
|---|---|---|
| **単一トークン** | GitHub / WordPress.com / Notion / Slack / Canva | 1 つの長い文字列を貼るだけ |
| **OAuth (Google アカウント)** | Google Drive / Calendar / Gmail | OAuth 画面でログイン → 表示されたコードを貼り付け |
| **メール + トークン + サイト URL** | Atlassian (Jira / Confluence) | 3 つの値をすべて入力 |

各サービスのページに「トークンの取り方」リンクが用意されているので、迷ったらそちらをご参照ください。

## 🆘 パスワードを忘れたら — リカバリーキーで復元

ご安心ください。**24 単語のリカバリーキーがあれば** パスワードを忘れても完全に復元できます。

復元の手順:

1. ロック画面の下にある **「パスワードを忘れた場合 — リカバリーキーで復元」** をクリック
2. 保管しておいた 24 単語を、スペース区切りで入力欄に貼り付け
3. **新しいマスターパスワード** を 2 回入力 (確認用)
4. 「復元してロック解除」を押す
5. すべての保存済みトークンがそのまま使える状態に戻ります

復元後は新しいパスワードでロック解除できるようになります。

> **現在の 24 単語は永続です** — Phase E v1 では再発行 / 入れ替えに対応していません。新しい mnemonic に切り替えたい場合は、ハードリセット (下の「万が一、両方とも失くしてしまった場合」の手順で実行できる **「すべてのデータを削除」**) を行ったうえで初期セットアップから再度やり直してください。
>
> なお、リカバリーキーによる復元 (`recoverWithMnemonic`) は旧 vault を v1 推奨仕様へ自動的にマイグレーションしますが、24 単語そのものは変わりません。

### 万が一、両方とも失くしてしまった場合

パスワードもリカバリーキーも失った場合、暗号化されたデータは **誰にも (Service Hub 開発者にも) 復号できません**。これは安全性の根幹を支える仕様で、バックドアは存在しません。

その場合は「最初からやり直す」ハードリセットを実行してください:

- ブラウザ版: 設定ページ → **「すべてのデータを削除して最初からやり直す」** ボタン
- もしくはブラウザの開発者ツールで IndexedDB を手動削除

リセット後、新しいパスワードで初期セットアップ画面からやり直せます (各サービスのトークンは再入力が必要です)。

## 🔒 自動ロックと安全性

Service Hub は、放置中の覗き見を防ぐため **自動でロック** されます。

- **タブを別画面に切り替えて 5 分** たつと自動ロック
- **キーボード / マウス操作がないまま 15 分** たつと自動ロック
- 手動でロックしたいときは、画面右上の **🔒 ロック** アイコンをクリック

ロックが掛かるとマスターパスワードを再入力するまで操作できません。
パスワード自体は **このデバイスから外に出ません**。すべてのトークンは **AES-GCM-256** という標準的な強力な暗号で守られています。これはオンラインバンキング等で使われているものと同じ強度の暗号方式です。

> ※ 自動ロックの時間 (5 分 / 15 分) は、現バージョンでは UI からの変更に対応していません。値を変えたい場合は、開発者向けに `src/renderer/security/autoLock.ts` 内の `DEFAULT_HIDDEN_MS` (タブ非表示時) と `DEFAULT_IDLE_MS` (無操作時) を編集し、再ビルドしてご利用ください。今後のバージョンで設定画面からの変更を予定しています。

## 📚 ライブラリで成果物を管理

「テンプレート」「チームレーダー」「事業ダッシュボード」などで作った成果物 (プレゼン HTML、Markdown レポート、SVG ロゴ、PNG 画像) は **すべてライブラリに自動保管** されます。

ライブラリ画面でできること:

- 直近順 / サービス別 / 検索でフィルタ
- **プレビュー** (HTML / SVG はそのまま表示、PNG は画像表示)
- **再ダウンロード** (PC のダウンロードフォルダへ)
- **Canva に送る** (新しいタブで Canva インポート画面を開く)
- **削除**

### 容量の上限

ライブラリの上限は **50 MB / 100 件** です。
上限を超えると、古いファイルから自動的に削除されます (新しい作品は常に保存されます)。重要な成果物はライブラリから定期的にダウンロードして PC やクラウドに保管してください。

## 🌐 CORS で動かないサービス — Cloudflare Worker プロキシ

ブラウザ版で **Notion / Atlassian / Cloudflare** の 3 サービスを使う場合のみ、「CORS」というブラウザのセキュリティ制限により **直接通信できません** (これは Service Hub の制限ではなく、それぞれの API 提供元の仕様です)。

解決策は「自分専用の小さな中継サーバー (プロキシ)」を 1 つだけ用意することです。**Cloudflare Worker** という無料サービスを使えば 5 分で立ち上がります。

ざっくりした手順 (詳細は `docs/PROXY_EXAMPLE.md` を参照):

1. Cloudflare のアカウントを作る (無料、クレジットカード不要)
2. ダッシュボードで **Create Worker** をクリック
3. 提供されている **30 行のサンプルコード** をコピーして貼り付け
4. **Deploy** ボタンを押す → `https://xxx.workers.dev` という URL が発行される
5. Service Hub の **設定 → BYO プロキシ** にその URL を貼り付けて保存

これだけで Notion・Atlassian・Cloudflare 連携が動くようになります。Cloudflare Worker の無料枠は 1 日 10 万リクエストまであるため、個人利用なら課金は発生しません。

デスクトップ版 (方法 B) ではプロキシ設定は不要です。

## ❓ よくある質問

**Q. パスワードを忘れたらどうなりますか?**
A. リカバリーキー (24 単語) があれば完全に復元できます。両方とも失くした場合は、暗号化されたデータは復号できないため、最初からやり直す (ハードリセット) ことになります。

**Q. 複数のデバイス (PC とノート PC) で同じデータを使えますか?**
A. 現状は同期機能がなく、データはそれぞれのデバイスのブラウザ / アプリ内に保管されます。それぞれでセットアップ・トークン入力が必要です (クラウド同期は将来検討中です)。

**Q. 入力したトークンや API キーは、Anthropic などの開発元に送られませんか?**
A. **いいえ、一切送られません。** トークンはお使いのデバイス内で AES-GCM-256 で暗号化され、各サービスの公式 API (api.github.com など) との通信にのみ使われます。Service Hub は中継サーバーを持ちません。

**Q. ブラウザ版とデスクトップ版で機能の違いはありますか?**
A. **63 サービスすべて両方で使えます。** 違いは内部の保管方法だけです。
- ブラウザ版: ブラウザ内蔵の暗号化ストレージ (IndexedDB + WebCrypto)
- デスクトップ版: OS の鍵管理 (Keychain / Credential Manager) + ファイル暗号化
- ブラウザ版では Notion / Atlassian / Cloudflare のみプロキシが必要

**Q. オフラインでも使えますか?**
A. ファイルを開く部分は動きますが、各サービスのデータ取得には当然インターネット接続が必要です。スナップショット (保存済みの過去データ) はオフラインでも閲覧できます。

**Q. データはどこに保存されていますか?**
A. ブラウザ版はブラウザのデータフォルダ (IndexedDB) 内。デスクトップ版は OS の userData フォルダ (例: macOS では `~/Library/Application Support/Service Hub/`) 内です。どちらも他のアプリやサイトからは読めません。
