## A. 見積依頼書（発注者 → 候補会社）

| 項目 | 内容 |
|---|---|
| 件名 | Service Hub 開発委託（対象フェーズ: ＿＿＿＿） |
| 見積範囲 | RFP §2.1 のスコープ。前提・除外事項は見積書に明記すること |
| 契約形態の希望 | 開発=請負（成果物完成責任）/ 保守・調査=準委任 を想定。代替案があれば提案可 |
| 見積有効期限 | 提出日から90日以上 |
| 提出期限 | ____年__月__日 |
| 提出先 | （メールアドレス等） |

### 見積の必須条件
1. **内訳を工程別・成果物別に分解**すること（一式◯円のみの見積は不可）。
2. 工数の根拠（人日・単価・スキルレベル）を示すこと。
3. 検収条件（[09_ACCEPTANCE.md](./09_ACCEPTANCE.md) の品質ゲート全グリーン）を織り込むこと。
4. 再委託を予定する場合はその範囲と金額を明示すること（事前承認制）。
5. 変更が発生した場合の追加見積単価（人日単価）を記載すること。
