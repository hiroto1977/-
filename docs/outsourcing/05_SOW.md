# 個別契約書（SOW）・注文書 様式

| 項目 | 内容 |
|---|---|
| 文書名 | 個別契約書 (Statement of Work) |
| 上位契約 | [業務委託基本契約書](./04_BASIC_CONTRACT.md)（本個別契約に共通適用） |

> 案件（フェーズ・機能群）ごとに本様式で締結する。請負形態の個別契約は印紙税に留意。

## 個別契約書

（発注者名）（甲）と（受託者名）（乙）とは、____年__月__日付業務委託基本契約書に
基づき、次のとおり個別契約を締結する。

### 1. 件名・契約形態

| 項目 | 内容 |
|---|---|
| 件名 | Service Hub ＿＿＿＿＿＿ 開発（例: Phase 6 live fetcher 拡張） |
| 契約形態 | ☐ 請負（成果物完成責任） ☐ 準委任（善管注意義務） |
| 契約期間 | ____年__月__日 〜 ____年__月__日 |

### 2. 委託業務の内容（スコープ）

| # | 業務 / 機能 | 参照仕様 |
|---|---|---|
| 1 | （例: GitHub live fetcher の Issues 対応） | specs/03 §◯、Issue #◯ |
| 2 | | |

**対象外（明示的除外）**: （例: 本番環境での結合確認、AIカウンセラー安全ルールの確定判断）

### 3. 成果物・納入

| # | 成果物 | 形式 | 納期 |
|---|---|---|---|
| 1 | ソースコード（マージ済み PR） | GitHub リポジトリ `main` への PR | ____年__月__日 |
| 2 | テストコード（品質ゲート全グリーン） | 同上（PR に同梱） | 同上 |
| 3 | 更新ドキュメント（ARCHITECTURE.md 同期含む） | 同上（PR に同梱） | 同上 |
| 4 | 作業報告書（準委任の場合） | PDF / Markdown | 月次 |

- 納入方法: 甲が指定するリポジトリへの Pull Request 提出をもって納入とする。
- 検収: [09_ACCEPTANCE.md](./09_ACCEPTANCE.md) の基準により、納入後 **10 営業日以内**に実施。

### 4. 委託料・支払

| 項目 | 内容 |
|---|---|
| 委託料（税抜） | ¥＿＿＿＿＿＿＿（内訳は ____年__月__日付 御見積書 No.＿＿ による） |
| 支払条件 | 検収合格後、請求書受領月の翌月末日払い（銀行振込・振込手数料は甲負担） |

### 5. 体制・連絡

| 役割 | 甲 | 乙 |
|---|---|---|
| 責任者 | | |
| 主担当 | | |

詳細は [07_STRUCTURE_RACI.md](./07_STRUCTURE_RACI.md) による。

### 6. 特記事項

1. 開発標準・品質ゲート・安全要件は基本契約第3条のとおり。
2. 仕様変更は [08_CHANGE_MANAGEMENT.md](./08_CHANGE_MANAGEMENT.md) の手続によらなければ
   効力を生じない（口頭合意のみによる変更は不可）。
3. 乙が業務上知り得た秘密情報・個人情報の取扱いは NDA および
   [06_SECURITY_REQUIREMENTS.md](./06_SECURITY_REQUIREMENTS.md) による。
