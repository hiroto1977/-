## 1. 情報資産の分類と取扱い

| 分類 | 例 | 委託先の取扱い |
|---|---|---|
| 機密 | 本番 API トークン・OAuth クライアントシークレット・ユーザーデータ | **提供しない**（受領も保持も禁止。発見時は即時報告・破棄） |
| 秘密 | ソースコード・設計文書・RFP・見積 | NDA の対象。アクセスは契約要員に限定 |
| 公開 | GitHub Pages 公開物（デモ HTML 等）・OSS 部分 | 制限なし |

- 開発・テストには**モック/ダミー認証情報のみ**を使用する（実トークン埋め込み禁止）。
- リポジトリへの秘密情報コミットは禁止。誤コミット時は即時報告し、履歴除去と
  当該情報の無効化（ローテーション）を甲と協力して行う。

## 2. 開発環境の要件

1. 作業端末は OS・ブラウザを最新に保ち、ディスク暗号化とスクリーンロックを有効化する。
2. リポジトリアクセスは甲が発行する権限に限る。アカウント共有禁止・多要素認証必須。
3. 要員の追加・離任時は速やかに甲に通知し、離任者のアクセス権を即日失効させる。
4. 私物端末・公共 Wi-Fi での作業可否は個別契約で定める（既定: 不可）。

## 3. コードのセキュリティ基準（CI で機械的に強制）

委託先は以下の自動検査の**全グリーン**を維持する義務を負う（検収条件の一部）。

| 検査 | 強制される内容 |
|---|---|
| `npm run lint:forbidden` | `nodeIntegration: true` / `contextIsolation: false` / `eval` / `innerHTML` 等の禁止パターン |
| `npm run lint:imports` | main / preload / renderer のプロセス境界違反の禁止 |
| `npm test` ほか | 秘密情報を renderer に渡さない設計（`window.serviceHub` ブリッジ経由のみ）の維持 |

加えて:
- 外部リンクは `serviceHub.openExternal` 経由のみ（`window.open` 禁止）。
- 新規依存パッケージの追加は甲の事前承認制（サプライチェーン統制）。
- 暗号実装（WebCrypto Vault・safeStorage）の方式変更は甲の書面承認を要する。

## 4. AIカウンセラー安全要件（最重要）

[specs/05 §7](../specs/05_AI_COUNSELOR.md) の安全境界を契約上の義務とする:

1. 危機検知マーカー・相談窓口情報・免責文言の変更は、**甲の書面承認なく行わない**。
2. 危機→窓口照会 100%・safety-miss 0 の**安全不変条件テストを削除・弱体化しない**。
3. 窓口・統計情報の追加は確証規律（独立2出典・公的1件以上）を満たす出典を添付する。

## 5. 個人情報・データ保護

1. 本システムはローカルファースト設計であり、委託先がユーザーの個人データを取り扱う
   ことは原則ない。受領が必要になる場合は事前に甲乙協議し、個人情報保護法および
   甲の指示に従う。
2. テストデータに実在の個人情報を使用しない（合成データのみ）。

## 6. インシデント対応

1. 秘密情報の漏洩・紛失・不正アクセスの発生またはその疑いを認知したときは、
   **24時間以内**に甲へ第一報を行い、その後の調査・是正・再発防止に協力する。
2. 報告には、影響範囲・原因・暫定対応・恒久対応（判明分）を含める。

## 7. 監査・点検

甲は、事前通知のうえ、委託先の本要件の遵守状況について報告を求め、または合理的な
範囲で監査を実施できる。重大な不遵守は基本契約第11条の解除事由となり得る。

## 8. 契約終了時

契約終了時、委託先は甲の秘密情報（コード・文書・アクセス情報）を NDA 第3条に従い
返還・破棄し、破棄証明を提出する。
