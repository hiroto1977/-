# 04. 申請ライフサイクル と リスク管理

## A. 共通ライフサイクル (補助金)

```
[1. 公募開始]
   ↓ (情報収集・適合判定)
[2. 申請書類準備]   ← 通常 1〜2 ヶ月
   ↓
[3. 公募締切・申請]
   ↓ (審査 1〜3 ヶ月)
[4. 採択発表]
   ↓
[5. 交付申請 → 交付決定]   ← この後でないと発注 NG
   ↓
[6. 事業実施]   ← 補助対象期間内に完了
   ↓
[7. 実績報告書 提出]
   ↓ (事務局による検査)
[8. 補助金額 確定]
   ↓
[9. 精算払い (入金)]
   ↓
[10. 事業化状況報告 (5 年間程度)]
```

### A-1. 各段階の典型的な所要時間

| 段階 | 所要 | 担当 |
|---|---|---|
| 1→3 | 1〜2 ヶ月 | 申請者 + 専門家 |
| 3→4 | 1〜3 ヶ月 | 事務局 |
| 4→5 | 数週間 | 申請者 |
| 6 | 数ヶ月〜1 年 | 申請者 |
| 7→9 | 1〜3 ヶ月 | 申請者 → 事務局 |
| 10 | 5 年間 | 申請者 |

合計: 申請開始から入金まで **6 ヶ月〜1 年半**。

## B. 助成金 (厚労省) のライフサイクル

```
[1. 計画届の作成・提出]
   ↓
[2. 計画期間に取組実施 (例: 正社員化、設備導入)]
   ↓
[3. 取組完了]
   ↓
[4. 賃金支払・対象者の継続雇用 (6 ヶ月等)]
   ↓
[5. 支給申請書 提出]
   ↓ (審査 数ヶ月)
[6. 支給決定 → 入金]
```

### B-1. 助成金の典型的所要時間

申請から入金まで **6 ヶ月〜1 年**。事業主の **支給申請書未提出 → 期限徒過 → 受給権喪失** が頻発するので **タスク管理必須**。

## C. 公庫融資のライフサイクル

```
[1. 事前相談 (公庫窓口 or 電話)]
   ↓
[2. 申込書類提出]
   ↓
[3. 担当者面談 (事業計画ヒアリング)]
   ↓ (場合により現地調査)
[4. 内部審査]
   ↓
[5. 融資決定 (or 否決)]
   ↓
[6. 契約締結 (印紙税負担あり)]
   ↓
[7. 入金]
   ↓
[8. 約定返済]
```

申込から入金まで **2 週〜2 ヶ月**。創業融資は審査が厚く時間長め。

## D. 民間融資 (銀行) のライフサイクル

```
[1. 事前相談]
   ↓
[2. 申込書類提出]
   ↓
[3. 信用情報確認・面談]
   ↓
[4. 審査 (本部承認)]
   ↓
[5. 保証協会審査 (協会保証付の場合)]
   ↓
[6. 契約 → 入金]
```

申込から入金まで **数日〜数週間** (取引深度・規模で大差)。

## E. リスク・チェックリスト (補助金)

### E-1. 申請段階

```
□ 公募要領を 3 回読んだ (要件・対象経費・スケジュール)
□ 自社が「対象事業者」に該当 (中小企業の定義、業種、資本金、従業員数)
□ 採択後の自己資金繰り (立替) を試算した
□ 補助対象経費と対象外経費を明確に分けた
□ 添付書類 (決算書・登記簿・納税証明 等) を揃えた
□ 電子申請の ID (G-BizID プライム 等) を取得済
□ 不採択時のプラン B がある
```

### E-2. 採択後・実施段階

```
□ **交付決定通知の日付以降に発注** (それより前は対象外!)
□ 見積書・契約書・納品書・支払証憑を全て紙＋電子で保管
□ 経費項目ごとに帳簿を分けた
□ 補助対象期間内に支払完了 (工事・サービス完了 + 支払 両方が期限内)
□ 計画変更が必要なら **事前に変更承認** を得る (事後申告は NG)
□ 中間検査・現地調査に対応できる体制
```

### E-3. 報告・精算段階

```
□ 実績報告書の提出期限 (採択後 X ヶ月以内、遅延即失格)
□ 経費の按分 (一部のみ補助対象の場合) を正しく計算
□ 消費税の取扱 (税抜計上 等) を理解
□ 関係書類を 5 年間保管 (税務 7 年と別建)
□ 調査・監査への対応窓口を社内決定
```

### E-4. 事業化状況報告

```
□ 報告期限を 5 年分カレンダー登録
□ 計画値 vs 実績値を期ごとに記録
□ **計画大幅未達** で「収益納付」(補助金返還) のリスク確認
```

## F. リスク・チェックリスト (助成金)

```
□ 計画届を **取組開始前に提出** したか
□ 就業規則の改定・労使合意の証跡
□ 助成対象者の選定が要件に合致 (年齢・雇用形態・期間 等)
□ 対象期間中の **解雇・自己都合退職** の発生リスク (発生で受給不可)
□ 事業所単位の **受給上限** (年度・通算)
□ 賃金台帳・出勤簿等の保管
□ 不正受給時の制裁 (返還 + 加算金 + 5 年間不支給 + 公表)
```

## G. リスク・チェックリスト (融資)

```
□ 月次資金繰り表 (24 ヶ月) を作成済
□ 借入後の毎月返済額が固定費に占める比率
□ 元本据置期間中の利息のみ支払い額
□ 連帯保証人 / 経営者保証 の有無と免除可能性 (経営者保証ガイドライン)
□ 期限の利益喪失事由 (不渡・信用不安 等)
□ コベナンツ (財務制限条項) の有無
□ 担保設定の対象資産・登記費用
□ 返済が滞った場合の代位弁済 → 信用情報 (ブラックリスト)
□ 返済原資の合理性 (営業利益 + 減価償却 で返済余力)
```

## H. 横断的なリスク

### H-1. 不正受給 (絶対 NG)

- 詐欺罪 (刑法 246 条)
- 補助金等適正化法 (補助金不正受給) — 返還 + **加算金 (年率 10.95% 等)** + **3 年〜5 年の不支給** + 公表
- 業界・専門家の信用喪失

**やってはいけないこと**:
- 売上・雇用者数の水増し
- 同じ経費の複数制度への重複申請
- 親族企業へのキックバック
- 領収書・契約書の改竄
- 計画と異なる用途への流用

### H-2. 「補助金倒れ」

採択 → 高額な設備投資 → 売上想定未達 → 借入金返済不能 → 倒産。
補助金は **既存事業を強化** する用途が安全。

### H-3. 専門家リスク

- 「採択請負人」による高額成功報酬 + 不正計画書作成
- 事務局審査で計画書のテンプレ的記述パターンが検出され不採択
- 採択後の事業遂行は **経営者本人** にしかできない

### H-4. 制度変更リスク

- 年度途中の予算枯渇 (公募終了)
- 後継制度名・要件の変更
- 政権交代等による政策方針転換

## J. デッドライン管理

`scripts/funding-deadline.sh` を週 1 回実行。**カレンダーアプリ** にも全期日を登録する習慣を。

主要マイルストーン例:
- 申請締切
- 交付決定後の発注期限
- 補助対象期間終了
- 実績報告書 提出期限
- 事業化状況報告 1〜5 年目
- 助成金の支給申請期限 (取組完了から X ヶ月以内が多い)
- 融資の毎月返済日

期限を 1 日でも超えると **権利消滅 = ゼロ円** になる制度がほとんど。
