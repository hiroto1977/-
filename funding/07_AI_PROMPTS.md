# 07. 申請書ドラフト用 AI プロンプト テンプレ

> ⚠️ 機密情報を含むプロンプトは **ローカル LLM (Ollama 等)** で実行。クラウド AI を使う場合は事前に PII / 営業秘密のマスキングを (`governance/05_TEMPLATES.md` A 節を参照)。

## A. 公募要領の論点抽出

```
あなたは中小企業診断士の補助業務をする AI です。
以下の補助金 公募要領から、申請書を書くために必要な「採点項目」「必須記載項目」「形式要件」「タイムライン」「対象経費」「対象外経費」を抽出して整理してください。

出力フォーマット:
1. 制度の主旨 (3 行)
2. 必須採点項目 (列挙 + 各項目の配点比重 推定)
3. 必須記載項目 (列挙)
4. 形式要件 (ページ数 / フォントサイズ / 添付書類)
5. タイムライン (公募開始 / 締切 / 採択 / 事業実施 / 報告)
6. 対象経費の分類
7. 対象外経費 (要注意)
8. 不採択になりやすいポイント (推定)

公募要領:
---
[ここに公募要領のテキスト or PDF 抜粋を貼る]
---
```

## B. 自社事業 → 公募要件 の適合度チェック

```
以下の (1) 自社事業概要 と (2) 補助金公募要領 の対応関係を分析し、
申請が妥当か、どのストーリーで書けば説得力が出るかを助言してください。

(1) 自社事業概要:
[業種、年商、従業員数、強み、計画している投資を 200 字程度]

(2) 公募要領 (要約):
[A 節の出力を貼る]

出力:
1. 適合度判定 (高 / 中 / 低 + 理由)
2. 強調すべき自社の強み (3 つ)
3. 弱点になりそうな箇所と対応案
4. 申請書のストーリー骨子 (起承転結)
5. リスク / 注意点
```

## C. 事業計画書 セクション ドラフト

```
事業計画書の「[セクション名]」を以下の制約で起草してください。

制約:
- 文字数: [N] 字 程度
- 読み手: [補助金審査員 / 銀行融資担当 / 投資家]
- トーン: 客観的・数値根拠あり・誇張なし
- 必ず含める要素: [箇条書き]
- 避ける表現: 「画期的」「絶対」「業界初」 等

入力情報:
[セクションに必要な事実・データ]

出力:
1. ドラフト本文
2. 不足情報の指摘 (これを埋めれば質が上がる項目)
3. 図表化したほうがよい部分の提案
```

## D. SWOT 分析

```
以下の自社情報から SWOT 分析を作成してください。

自社情報:
[業種、規模、製品/サービス、地域、顧客層、競合、財務概要]

外部環境:
[業界トレンド、法規制、技術革新、社会情勢]

出力:
1. S (強み) / W (弱み) / O (機会) / T (脅威) の表
2. クロス SWOT 戦略 (SO / WO / ST / WT)
3. 最も注力すべき戦略 1 つと根拠
```

## E. 競合分析

```
以下の競合 [N] 社について、項目別に比較表を作成してください。

比較項目: 価格・品質・納期・サービス範囲・強み・弱み・想定戦略

競合情報:
- [社名 1]: [入手可能な公開情報を貼る]
- [社名 2]: ...

出力:
1. 比較表 (Markdown)
2. 自社の差別化余地
3. 各競合の動き方の予測
4. 不足情報のリスト (公開情報から取れない項目)

注意: 推測は「(推定)」と明記し、断定しない。
```

## F. 売上計画 数値妥当性チェック

```
以下の売上計画について、数値の妥当性を批判的にレビューしてください。

入力:
- 業種: [業種]
- 立地: [都市名 / 地域]
- 商品/サービス: [説明]
- ターゲット: [ペルソナ]
- 計画売上 (3 年):
  Year 1: ___ 万円
  Year 2: ___ 万円 (前年比 ___%)
  Year 3: ___ 万円 (前年比 ___%)
- 想定客数: [数値]
- 想定客単価: [数値]
- 計算根拠: [自分の算出ロジック]

出力:
1. 業界一般的な成長率と比較しての妥当性
2. 客数 × 単価 が現実的か
3. 同業類似ケースから推定する範囲との乖離
4. 仮説に「もし○○なら○○」を加えるべき箇所
5. 修正提案
```

## G. リスク棚卸し

```
以下の事業計画について、想定されるリスクを網羅的に洗い出してください。

事業概要:
[200 字]

カテゴリ別に:
1. 市場リスク (需要・競合)
2. オペレーションリスク (生産・物流・品質)
3. 財務リスク (資金繰り・為替・金利)
4. 法務・規制リスク
5. 人材リスク (採用・離職)
6. IT / 情報セキュリティ
7. 自然災害・パンデミック
8. 評判・SNS

各リスクに:
- 発生確率 (高 / 中 / 低)
- 影響度 (大 / 中 / 小)
- 想定される対応策 (予防 / 軽減 / 移転 / 受容)
```

## H. 融資面談 想定問答

```
公庫担当者との面談を想定し、以下の事業計画について 想定質問 と 模範回答 を作成してください。

事業計画 (要点):
[300 字]

想定質問カテゴリ:
1. 経営者の経歴・覚悟
2. 事業の独自性・競合優位性
3. 売上見込みの根拠
4. 返済計画の妥当性
5. 想定外シナリオへの対応
6. 既存借入の状況
7. 個人保証 / 担保
8. 自己資金の出所

各質問に:
- 質問の意図 (担当者は何を見極めたいか)
- 模範回答の骨子 (200 字)
- やってはいけない回答
```

## I. 補助金 不採択理由の分析

```
以下は 私が提出した申請書の概要 と 不採択通知 (理由) です。
次回の再申請に向けて改善ポイントを整理してください。

申請書概要:
[5 行]

不採択理由:
[通知の引用]

過去の採択事例 (公開分):
[J-Net21 等で取得した同種制度の採択事例タイトル数件]

出力:
1. 理由の解釈 (本当に問われているのは何か)
2. 申請書の弱点推定
3. 改善案 (セクション別)
4. 次回までに収集すべきデータ・実績
5. 再申請時期の提案
```

## J. 専門家依頼書ドラフト

```
中小企業診断士 (or 社労士 / 税理士 / 弁護士) に依頼する際の
依頼書骨子を作成してください。

依頼の目的: [補助金申請支援 / 助成金 / 融資交渉 / 顧問契約 等]
自社情報: [社名・業種・規模]
具体的な相談事項:
1. ___
2. ___
3. ___
予算感: [ある or なし]
希望スケジュール: [いつまでに何を]

出力:
1. メール / 文書ベースの依頼書ドラフト
2. 専門家との面談時に持参すべき資料リスト
3. 依頼前に自社で整えておくべき情報
4. 報酬体系の質問項目
```

## L. 成果物の検証プロンプト (出力後)

```
以下は AI が起草した [文書の種類] です。
公開・提出する前に、次のチェックを行ってください。

入力:
[AI 出力]

チェック項目:
1. 事実誤認の有無 (固有名詞・数値・引用元)
2. 公募要領の必須記載事項を満たしているか
3. 矛盾する記述がないか
4. 不必要な誇張表現がないか
5. 想定読み手 (審査員 / 銀行員) に届くか
6. 機密情報の漏洩 (不要な数値の記載) はないか
7. 法的に断定的な記述 (「合法」「違法」等) はないか

出力:
- チェック結果 (✅ / ⚠️ / ❌ + 該当箇所)
- 必須修正事項
- 推奨修正事項
- 提出前の最終確認手順
```

## M. プロンプト使用時の標準ワークフロー

```
1. [ローカルで] 公募要領を A プロンプトで論点抽出
   ↓
2. B プロンプトで自社適合度判定
   ↓
3. 経営者本人が ストーリー骨子を決める (AI 不可)
   ↓
4. C プロンプトで セクション別ドラフト
   ↓
5. F プロンプトで 数値妥当性チェック
   ↓
6. G プロンプトで リスク棚卸し
   ↓
7. L プロンプトで 自己検証
   ↓
8. 専門家レビュー (J プロンプトで依頼書作成)
   ↓
9. 修正・最終化
   ↓
10. 提出
```

すべて **個別案件 = ローカル LLM**、**業界一般情報 = クラウド AI 可** の使い分けで。
