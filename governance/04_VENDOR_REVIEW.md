# 04. クラウド AI ベンダ評価チェックリスト

クラウド AI を採用する前に、最低限確認すべき項目。「採用済み」のサービスも年 1 回見直し。

## A. 契約・規約レイヤ

| 項目 | 確認 | NG なら |
|---|---|---|
| 利用規約に「学習データとして利用する」条項があるか | 個人版・無償版にはほぼある | 業務利用しない |
| API/Enterprise 版で **学習に使わない** ことが明文化されているか | DPA 等で確認 | 別ベンダ検討 |
| データ保持期間 (請求後即削除/30日/無期限) | 短いほど安全 | 用途を限定 |
| データ処理者 (Processor) としての義務を負うか | DPA の有無 | 個人データ送信不可 |
| 越境移転先 (米/EU/その他) | APPI 28 条対応 | 同等水準 OR SCC OR 同意 |
| 違反時の損害賠償上限 | 一般に低い (年額の数倍) | 重要案件は使わない |
| **責任分界** (顧客の責で起きた漏えいは免責) | 通常そう書かれている | 自社が一次責任 |
| **準拠法・裁判管轄** | 海外裁判管轄なら実効性に難 | 重要案件は要交渉 |

## B. セキュリティレイヤ

| 項目 | 確認 |
|---|---|
| 認証方式 (パスワード/MFA/SSO) | MFA 必須化可能か |
| **送信時暗号化** (TLS 1.2 以上) | 通常満たす |
| **保存時暗号化** | AES-256 等 |
| 顧客管理鍵 (CMK / BYOK) サポート | 高機密用途で重要 |
| アクセス監査ログのエクスポート可否 | コンプライアンス必須 |
| 第三者監査 (SOC 2 Type II / ISO 27001 / ISMS) | 取得済みか |
| 脆弱性開示プロセス (PSIRT/responsible disclosure) | 公開しているか |
| インシデント通知 SLA (発生から何時間以内に顧客通知) | 24h 〜 72h が標準 |

## C. データ取り扱いレイヤ

| 項目 | 確認 |
|---|---|
| プロンプトのレビュー (人間目検) はあるか | レビューありなら高機密データ送信不可 |
| 出力フィルタリング (検閲・モデレーション) ログ | 内容が記録されることがある |
| 第三者 (サブプロセッサ) リスト公開 | 存在するか |
| データ削除リクエストへの対応 SLA | 30 日以内が標準 |
| データポータビリティ (エクスポート機能) | ロックイン回避 |
| サービス終了時のデータ取扱 | 通知期間と返却・破棄方針 |

## D. AI 固有レイヤ

| 項目 | 確認 |
|---|---|
| モデルが定期更新される場合の挙動変化通知 | 通知 / 旧版固定可否 |
| プロンプトインジェクション耐性 | ベンダの対策状況 |
| 出力のハルシネーション率の公開情報 | 業務リスクの判断材料 |
| 商用利用の可否・条件 | 生成物の権利帰属 |
| 利用料金の予測可能性 | トークン課金の上限設定 |

## E. 運用継続レイヤ

| 項目 | 確認 |
|---|---|
| 稼働率 SLA (99.9% 等) | 業務クリティカルなら 99.9%+ |
| 障害時の自動フェイルオーバー | あれば |
| 価格改定の予告期間 | 30 日以上が標準 |
| サポート窓口 (日本語/英語/24h) | 業務時間カバー |
| ベンダ事業継続性 (買収・撤退リスク) | 財務情報・最近のニュース |

## G. 採用判定 シンプル版

```
クラウド AI 採用するなら最低限:
□ 法人版/API 版で、学習 opt-out が明文
□ 第三者認証 (SOC2 / ISO 27001) を取得
□ DPA を結ぶ (要事業者交渉、個人事業主は約款型でも可)
□ インシデント通知 SLA がある
□ データ保持期間が 30 日以下、または短縮可
□ 撤退時のデータ取扱が規定されている

これらを満たせない場合: ローカル LLM のみで対応する
```

## H. 個別案件の判定フロー

```
入力データを準備 →
  ↓
データクラスを判定 (C0〜C4)
  ↓
C3 以上 → ローカル LLM へ
C2 → A 〜 G を満たすベンダなら可、満たさなければマスキング
C0/C1 → 通常クラウド AI で可
```
