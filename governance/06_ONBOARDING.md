# 06. オンボーディング手順

新メンバー (社員・業務委託・インターン等) が AI 支援業務に着手するまでのチェックリスト。

## A. 着任 1 日目

### A-1. 説明と誓約

```
□ ガバナンス文書の存在と読む順番を案内
   - governance/README.md
   - 02_DATA_CLASSIFICATION.md (最優先)
   - 03_OPERATIONS.md
□ 秘密保持誓約書の署名 (退職後 X 年の存続条項を含む)
□ AI 利用に関する個別同意書 (社内規程による)
□ 個人情報保護法・不正競争防止法の概要を 30 分で説明
```

### A-2. 物理セキュリティ

```
□ 業務用 PC の貸与 (個人の私物 PC 利用は原則不可)
□ 端末暗号化の有効化 (FileVault / BitLocker / dm-crypt)
□ パスワードマネージャ (1Password / Bitwarden 等) のセットアップ
□ MFA (Authy / Aegis / yubikey) のセットアップ
□ スクリーンロック自動化 (5 分以内)
□ 物理的覗き見防止 (在宅は背後の人通り、職場はプライバシーフィルタ)
```

## B. 環境セットアップ

### B-1. 共通

```
□ git のセットアップ + 本リポジトリのクローン
□ python3 (静的サーバー用)
□ scripts/preflight.sh の実行 → 全 ✅ 化
□ scripts/pii-scan.sh の使い方の理解 (commit 前のローカル検査用)
```

### B-2. ローカル LLM

```
□ Ollama のインストール
□ OLLAMA_ORIGINS=* の設定 (またはピン留め)
□ モデル取得: llama3.2 (最低限) / qwen2.5 / llama3.2-vision
□ v19 ダッシュボードで「接続テスト」成功
□ ローカル専用モードのオン/オフ確認
```

### B-3. クラウド AI (扱う案件で必要なら)

```
□ 法人契約のアカウント発行 (個人アカウント不可)
□ 学習 opt-out の設定済確認
□ 与えられる API キーの保管方法 (パスワードマネージャ)
□ 04_VENDOR_REVIEW.md の該当ベンダ章の読了
```

## C. 着任 1 週目: 学習タスク

### C-1. データ分類クイズ

`02_DATA_CLASSIFICATION.md` のケース集 50 件のうち 10 件をランダム出題、自力で C0〜C4 を判定。担当者と答え合わせ。

### C-2. テンプレ実行訓練

`05_TEMPLATES.md` の以下を実際に試す:

```
□ A-1 マスキング → ローカル LLM に投入
□ B   データクラス判定
□ D   議事録要約 (架空データで)
□ E   ビジネスメール下書き
```

### C-3. インシデント対応シミュレーション

口頭で以下の状況を提示し、対応を確認:

```
ケース 1: 顧客リスト (氏名・メール) を ChatGPT 個人版に貼ってしまった
ケース 2: 業務委託先の担当者がメール添付で機密ファイルを社外個人 Gmail に送った
ケース 3: 社内チャットで取引先名と価格をうっかりオープンチャネルに投稿
```

期待される回答: 03_OPERATIONS.md G 節の対応手順を 3 分以内に説明できる。

## D. 着任 1 ヶ月目

```
□ 担当案件のデータクラスを管理者と確認
□ 平時のチェックリスト (05 の J/K) を毎日実施できている
□ AI 利用記録 (05 の I) を 1 週間分残している
□ 不明点の質問先 (Slack チャンネル / 担当者) を把握
```

## E. 6 ヶ月レビュー

```
□ ガバナンス文書の改訂版を再読
□ 利用ツールの棚卸し (使わなくなったクラウド AI のアカウント解除)
□ パスワード・API キーのローテーション
□ ヒヤリハット事例の共有
```

## F. 退職・契約終了時

```
□ 03_OPERATIONS.md L 節の退職手順を実施
□ 業務 PC の返却 (データ抹消の証跡を記録)
□ アカウント停止リストとの照合
□ 紙資料の返却・破棄
□ 秘密保持契約の存続を再確認
```

## G. オンボーディング担当者向けノート

- 「ローカル LLM 既定」の文化を最初に刷り込む
- 「迷ったら 1 段階上のクラスで扱う」を口癖にする
- 失敗例を共有しやすい雰囲気作り (隠蔽が最大のリスク)
- 月 1 回 30 分のガバナンス勉強会を設定

## H. 業務委託先 (フリーランス・外注会社) 向けの追加項目

```
□ 業務委託契約書に AI 利用条項を含めたか
   - 利用可能な AI ツール (ホワイトリスト)
   - 入力可能なデータクラス上限
   - 学習目的での利用禁止
   - 出力物の権利帰属
   - 違反時の損害賠償
□ 実作業端末がガバナンスを満たすか確認
□ 完了時のデータ削除証跡の取り付け
```

## I. 子供・家族との情報共有 (在宅ワーカー向け)

家庭内でも:

```
□ 業務用端末は家族と共有しない
□ 業務電話・Web 会議の音声が家族に漏れない位置取り
□ 印刷物は子供の手の届かない場所
□ 自宅 Wi-Fi の WPA3 化、推測されにくいパスワード
```

家族の善意でも C2 以上の情報の不用意な拡散はインシデント。
