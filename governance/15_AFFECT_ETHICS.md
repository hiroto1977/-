# 15. Affect-Aware Adaptive Chat — 倫理ガード と 設計

> **目的**: ユーザーの発話から推定した「感情の次元」に応じて応答スタイルを適応させる機能 (`v19/ui/dashboard.js: affect-aware mode`) の **倫理境界・透明性・ユーザー権利** を明示する。
>
> **対象**: 機能を有効化する全ユーザー、レビュア、AI 支援者
> **読了時間**: 5 分

---

## 0. 一行 サマリ

このシステムは **感情の次元** (快/不快、鎮静/興奮、急ぎ/余裕、砕け/丁寧) を heuristic に推定して応答スタイルを調整する。**性別・年齢・民族・地域 など protected attributes での分類は実装上 禁止** され、診断/医療目的では使えない。

---

## 1. やること / やらないこと

### 1.1 やること
- 4 次元の連続値 (PAD-like) で発話の「**温度感**」を推定
- その温度感に応じて応答スタイル (短く / 共感的 / 論理的 等) を毎ターン切替
- 推定値 を **ユーザーに見える** (UI 上に表示)
- ユーザーの **オプトイン 必須** (設定 → 既定 OFF)

### 1.2 **絶対やらないこと** (実装上禁止)
| 禁止項目 | 理由 |
|---|---|
| 性別 (男 / 女 / 他) で分類 | 科学的根拠なし、APPI 要配慮個人情報、EU AI Act 抵触 |
| 年齢層で分類 | 年齢差別、文化差・個人差の方が大きい |
| 民族・国籍・宗教で分類 | 差別的 profiling、いかなる目的でも禁止 |
| 性的指向で分類 | 同上 |
| 政治的志向で分類 | 同上 |
| 障害の有無を推定 | 医療情報、専門家領域 |
| **これらを発話から推定する識別器** | 学習データ自体が biased、推定誤差が個人を傷つける |

実装側のチェック: `tests/js/test_affect.mjs` で「protected-attribute marker (性別関連語等) を含む入力でも分類結果が同一 か」を確認。

---

## 2. 推定する 4 次元 (PAD + α)

| 次元 | レンジ | 意味 | 推定 ヒント |
|---|---|---|---|
| **valence** | 0.0 (不快) ~ 1.0 (快) | 感情の質 | 否定/肯定 マーカ語 |
| **arousal** | 0.0 (鎮静) ~ 1.0 (興奮) | 活性度 | `!` 数 / 反復文字 / 絵文字 |
| **urgency** | 0.0 (余裕) ~ 1.0 (急ぎ) | 時間圧力 | 「急ぎ」「すぐ」「ASAP」 |
| **formality** | 0.0 (砕け) ~ 1.0 (丁寧) | 距離感 | 敬語 / 文末助詞 |

各次元は **独立** に推定し、互いに影響させない (組合せから「人格」を推定するな)。

---

## 3. 透明性 (Transparency)

### 3.1 ユーザーが見える
- 設定タブで 4 次元の **現在値** が常時表示
- 各次元クリックで「なぜこの値?」短い説明 (どのキーワード/特徴が効いたか)
- セッション履歴: 直近 10 ターンの軌跡 (折れ線)

### 3.2 ユーザーが操作できる
- マスター トグル: 機能 ON/OFF (既定 OFF)
- 個別オーバーライド: 「私の今の formality は 0.9 だと思って」と手動設定
- 履歴削除: ボタン 1 つで全 affect データを 即時 wipe
- 全体無効化: トグル OFF で 推定処理自体が走らなくなる

### 3.3 ブラックボックス禁止原則
推定ロジックは heuristic ベース (透明)。重み・閾値は `dashboard.js` ソースに明記。
ML モデルや LLM に分類を委ねない (説明可能性を維持するため)。

---

## 4. データ保護 (APPI / GDPR 整合)

| 項目 | 実装 |
|---|---|
| **同意** | オプトイン トグル + 初回 ON 時に 確認 ダイアログ |
| **目的限定** | 応答スタイル 調整のみ。他用途禁止 |
| **保存場所** | ブラウザ localStorage のみ。外部送信ゼロ |
| **保存期間** | セッション間で保持するか は別オプション (既定: セッション終了で消去) |
| **削除権** | ボタン 1 つで即時消去 |
| **アクセス権** | 設定タブ で全データを JSON エクスポート可 |
| **第三者提供なし** | データは端末から出ない |
| **監査** | `audit.jsonl` に「分類器が走った」事実のみ記録 (推定値そのものは記録しない) |

データ クラス: **C3 (機密)** として扱う。

## 5. 制約と免責

### 5.1 これは **○○ ではない**
- ❌ 心理学的診断
- ❌ 医療デバイス
- ❌ うつ病/不安障害 のスクリーニング
- ❌ パーソナリティ評価
- ❌ 採用/評価 の判断材料
- ❌ 法的紛争での証拠

### 5.2 何ができるか
- ✅ 応答スタイルを発話の温度感に合わせる (ユーザー体験向上)
- ✅ 急ぎの問合せに 短く即答する判断材料
- ✅ 砕けた会話で過剰に丁寧にならない調整
- ✅ ユーザーが自分の発話の「読まれ方」を可視化

### 5.3 既知の限界
- heuristic ベースなので皮肉/ジョーク/方言 の誤分類あり
- 日本語ベースの語彙チューニング、他言語 では精度が下がる可能性
- 文化・個人差 を平均化してしまう恐れ → 4 次元値はあくまで参考

## 6. EU AI Act / 日本 AI 規制 整合

| 規制 | このシステムの対応 |
|---|---|
| EU AI Act 第 5 条 (禁止される実務) | **職場・教育コンテキストでは使わせない**。利用規約で明示 (将来) |
| EU AI Act 高リスク AI | 雇用/評価/重要 サービス で使わない (用途限定) |
| 日本 AI 事業者ガイドライン | 透明性・説明責任・人間の関与 を満たす設計 (§3 透明性) |
| APPI 第 17 条 (要配慮個人情報) | 性別/民族 等を扱わないことで適用外 |
| APPI 第 27 条 (第三者提供) | 第三者提供しないので適用外 |

## 7. ユーザーへの 表示文言 (UI)

オプトイン トグル説明:

```
🎭 感情適応モード (試験機能)

ON にすると、あなたの発話から「快/不快」「鎮静/興奮」「急ぎ/余裕」
「砕け/丁寧」の 4 次元を heuristic に推定し、応答スタイルを調整します。

・推定値は本端末内 (localStorage) のみで処理、外部送信なし
・性別・年齢・民族 等での分類は行いません
・診断/医療目的では使えません
・履歴はいつでも削除できます

詳細: governance/15_AFFECT_ETHICS.md
```

初回 ON 時の確認:
```
⚠️ 感情の推定を有効化しますか?

これは試験機能で、心理学的診断ではありません。あくまで応答スタイル
の自動調整のためのみに使われ、推定結果はあなたにも常時表示されます。

[キャンセル] [理解して有効化]
```

## 8. 実装上の チェックリスト (β/γ への引継ぎ)

- [ ] 既定で OFF
- [ ] 4 次元すべて 0.0 ~ 1.0 に clamp
- [ ] 推定根拠 (どのマーカが効いたか) を即座に説明可能
- [ ] 性別/年齢 マーカを含むが他が同一の入力 → 同じ分類値が出る (drift 検出テスト)
- [ ] 推定値は localStorage のみ、audit.jsonl には事実のみ
- [ ] OFF にしたら識別器自体が走らない (副作用ゼロ)
- [ ] 削除ボタンで affect 関連データのみ消える (チャット履歴は残す)
- [ ] 単体テスト: 強い正/負 マーカで valence が 0.3 / 0.7 を超える等
- [ ] 既存 INV を一切壊さない (smoke-test 全 通過)
