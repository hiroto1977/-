# SNS 炎上 対応フロー

## A. 覚知 (アラート発火 〜 1 時間)

```
□ 拡散範囲の把握 (リプライ数 / リポスト数 / 言及主体)
□ 炎上の事実確認 (誤情報なら即訂正、事実なら向き合う)
□ 関係者全員のSNS発言ストップ (個人アカウントも)
□ 経営層 + 広報 + 法務 緊急召集
□ Wayback Machine 等で原投稿の保全
```

## B. 短期対応 (24 時間以内)

### B-1. 事実確認チェックリスト
```
□ 何が問題視されているか (5 W 1 H)
□ 弊社に責任があるか (有 / 無 / 一部)
□ 関係する社員 / 取引先 / 顧客
□ 同種事案の過去事例
```

### B-2. 公式ステートメント案
```
[事実認識] 弊社は [事象] を認識しております。
[謝罪 / 説明] [責任ある場合は明確に謝罪、誤解なら丁寧に説明]
[対応] 現在 [調査中 / 対応中]。詳細は [N] 日以内に改めてご報告いたします。
[ご迷惑への姿勢] 関係者の皆様にご心配をおかけしておりますことを深くお詫び申し上げます。

[YYYY-MM-DD]
[社名] [代表者]
```

### B-3. 公開チャネル
```
□ 公式サイト プレスリリース欄
□ 公式 SNS アカウント
□ 顧客 個別連絡 (重大顧客)
□ 取引先 個別連絡
```

## C. 中期対応 (1 週間以内)

```
□ 詳細調査結果の公表
□ 原因分析 + 再発防止策
□ 必要なら関係者処分 (公表は慎重に)
□ 被害者がいる場合の補償提案
□ 取材対応窓口の一本化
```

## D. やってはいけないこと

```
× 反論 / 言い訳から始める
× 個人アカウントでの感情的発信
× 黙殺・削除して逃げる (Wayback などで保全されている)
× 法的対応をいきなり前面 (脅しに見える)
× 反対意見への一方的ブロック
× 関係者への口止め強要
```

## E. 法的対応の判断

```
□ 名誉毀損・業務妨害の証拠保全 (URL + 日時 + キャプチャ)
□ プロバイダ責任制限法に基づく発信者情報開示請求
□ 民事提訴 (損害賠償) は最終手段
□ 刑事告訴 (脅迫・業務妨害)
□ 弁護士相談必須
```

## F. 平時の備え

```
□ SNS 運用ルール (誰が何を発信できるか)
□ 私物アカウントでの会社言及ガイドライン
□ 危機対応 体制図 (経営者 / 広報 / 法務 / 顧問弁護士)
□ 危機対応 マニュアル (本書) の年 1 回見直し
□ 訓練 (年 1 回 模擬炎上演習)
□ メディアトレーニング (経営者の話し方)
```

## G. 関連法令

- プロバイダ責任制限法 (発信者情報開示)
- 不正競争防止法 (営業誹謗)
- 著作権法 (引用・スクショ)
- 個人情報保護法 (謝罪文での個人情報開示)
- 名誉毀損罪 (刑法 230 条)
- 信用毀損・業務妨害罪 (刑法 233 条)
