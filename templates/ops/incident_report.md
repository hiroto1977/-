# インシデント報告書 ひな形

```
インシデント ID: [INC-YYYYMMDD-NN]
報告日時: [YYYY-MM-DD HH:MM]
報告者: [氏名]
重大度: [ ] 高 (顧客影響大) / [ ] 中 / [ ] 低

== 1. 概要 (3 行で) ==
[何が / いつ / どんな影響]

== 2. 発生・覚知 ==
発生時刻 (推定): [YYYY-MM-DD HH:MM]
覚知時刻: [YYYY-MM-DD HH:MM]
覚知経緯: [監視アラート / 顧客連絡 / 社員発見 等]

== 3. 影響範囲 ==
影響を受けた顧客 / システム / データ:
- [ ]
影響を受けた時間: [HH:MM 〜 HH:MM] (合計 [ ] 分)
影響件数: [ ] 件 / [ ] 名

== 4. 時系列 ==
| 時刻 | 出来事 | 対応 |
|---|---|---|
| HH:MM | | |
| HH:MM | | |

== 5. 直接原因 ==
[何が技術的・運用的に起きたか]

== 6. 根本原因 (Why × 5) ==
1. なぜ発生? → [ ]
2. なぜそうなった? → [ ]
3. ...
5. 根本: [ ]

== 7. 応急対応 ==
[拡大防止のため何を行ったか]

== 8. 再発防止策 ==
- 短期 (1 週間以内):
- 中期 (1 ヶ月以内):
- 長期 (3 ヶ月以内):

== 9. 関係者対応 ==
- 顧客通知: [済 / 未]
- 個情委通知: [対象 / 対象外]
- 警察通知: [対象 / 対象外]
- 社外公表: [対象 / 対象外]

== 10. 関連文書 ==
- ログ: [リンク / 場所]
- スクリーンショット: [ ]
- 関連 SOP: [ ]

== 11. 教訓 ==
[他チーム / 他案件への横展開ポイント]

承認: [上位管理者 氏名] [YYYY-MM-DD]
```

## 注意
- データクラス: 内容により C3 〜 C4
- 個人情報漏えいの場合は `governance/03_OPERATIONS.md` G 節 (法的報告期限 3〜5 日 / 30 日) を参照
- インシデント情報自体が機密 → 共有範囲を限定
