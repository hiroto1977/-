# 提案書 ひな形 (10〜15 ページ想定)

```
[案件名] ご提案書
[YYYY-MM-DD]
[社名] [担当者]
ご提案先: [顧客名] [ご担当者] 様

== 表紙 ==
案件名 / 提案先 / 提案者 / 日付

== Executive Summary (1 ページ) ==
- ご提案の要点 (3 行)
- 期待される効果 (定量 + 定性)
- 投資額 / 期間 / 体制
- 次のアクション

== ご提案の背景 ==
- お客様が直面している課題 (ヒアリングからの理解)
- 課題が解決された状態のイメージ
- 弊社がご提案する理由

== 提案内容 ==
1. ソリューション概要
2. 機能 / サービス内容
3. 導入後のあるべき姿
4. (図解)

== 実施計画 ==
- フェーズ分け
- スケジュール (Gantt)
- マイルストーン
- 体制図 (お客様 + 弊社)

== 期待される効果 ==
- 定量効果 (売上 / コスト / 工数)
- 定性効果 (品質 / 満足度 / 競争力)
- 算出根拠

== お見積 ==
- 初期費用
- ランニング (月額 / 年額)
- 内訳 (作業項目別 + 諸経費)
- お支払条件

== 弊社のご紹介 ==
- 会社概要
- 主要実績 (類似案件)
- 担当チーム

== Q & A 想定 ==
- 想定質問 + 回答 (3〜5 件)

== 次のアクション ==
- ご検討期間
- ご決定後のスケジュール
- 契約形態
```

## 提案書 作成のコツ
- **顧客の課題から始める** (自社サービスの説明から始めない)
- 数字は出典を明記
- 競合との比較は事実ベース
- 失注時の振り返りを次案件に活かす (`templates/sales/lost_deal_review.md` で記録)
